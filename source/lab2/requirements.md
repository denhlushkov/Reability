# Requirements
---

- **FR01**: The system must allow the Patient to view the Therapist's schedule and book a session.

- **FR02**: The system must automatically verify the availability of slots before confirming a booking.

- **FR03**: The system must allow the Patient to view their schedule of past and future sessions.

- **FR04**: The system must provide the Therapist with the functionality to record clinical notes in the Patient's medical record.

- **FR05**: The system must require the Therapist to check the Patient's history before adding new records.

- **FR06**: The system must allow the Therapist to create an Invoice for a completed session.

- **FR07**: The system must provide the Patient with the option to pay the invoice online.

- **FR08**: The system must automatically update the session status (e.g., to "Awaiting Payment" or "Paid") following key financial events.

- **FR09**: The system must automatically send notifications to the Patient and/or Therapist regarding bookings, cancellations, or status changes.

- **FR10**: The system must allow the Patient to view the invoice details prior to payment.
---
- **NFR01**: The response time for Managing Therapist Availability must not exceed 3 seconds.

- **NFR02**: All Patient Data Recording and Transaction Processing must comply with relevant industry security standards.
